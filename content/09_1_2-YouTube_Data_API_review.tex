This section is based primarily on the official YouTube Data API documentation provided by Google for developers \cite{youtube_data_api_docs}. The documentation is used as the main source to describe the API functionality, available endpoints, request parameters, quota management and documented limitations.

\subsubsection{Functionality and general overview}

The YouTube Data API v3 is the official interface provided by Google for accessing and, in some cases, managing YouTube data from external applications. It allows developers to interact with structured resources represented in JSON format, including channels, videos, playlists, search results, comments and comment threads. These resources are exposed through resource-specific methods such as \texttt{list}, \texttt{insert}, \texttt{update}, and \texttt{delete}, depending on the type of resource and the permissions required \cite{youtube_data_api_docs}.

To use the API, developers must create a Google project, enable the YouTube Data API, and provide valid credentials with each request. In general, requests for public data can often be made with an API key, while requests that modify data or access private user information require OAuth 2.0 authorization. Another important characteristic of the API is its support for partial resource retrieval. The \texttt{part} parameter is required to specify which top-level sections of a resource should be returned, and the \texttt{fields} parameter can be used to reduce the response further by selecting only the nested properties needed by the application. This design improves efficiency by reducing unnecessary data transfer and processing. In addition, the API operates under a quota system, so requests must be designed carefully, especially in research-oriented data collection workflows \cite{youtube_data_api_docs}.

\subsubsection{Available endpoints and methods}

The YouTube Data API v3 is organized by resource type rather than as a single generic endpoint. Each resource exposes one or more methods according to the operations supported for that entity. In general, the API provides methods such as \texttt{list}, \texttt{insert}, \texttt{update}, and \texttt{delete}, although not all resources support all of these operations. This resource-oriented structure makes it possible to separate data retrieval, content creation, modification and administrative actions according to the needs of each application \cite{youtube_data_api_docs}.

For the purposes of this project, the most relevant resources are \texttt{search}, \texttt{videos}, \texttt{channels}, \texttt{commentThreads}, \texttt{comments}, \texttt{playlists}, and \texttt{playlistItems}. The \texttt{search} resource supports the \texttt{search.list} method, which returns search results that ma~tch the specified criteria. However, these results do not represent persistent resources themselves; instead, they point to videos, channels, or playlists that should later be queried through their corresponding resource-specific methods in order to obtain more detailed metadata \cite{youtube_data_api_docs}.

The \texttt{videos} resource provides methods such as \texttt{videos.list}, \texttt{videos.insert}, \texttt{videos.update}, and \texttt{videos.delete}, in addition to other video-related operations. The \texttt{channels} resource supports \texttt{channels.list} and \texttt{channels.update}. In the context of this thesis, \texttt{videos.list} is especially important because it allows the retrieval of structured metadata for one or more videos, while \texttt{channels.list} can be used to obtain channel-level information and related playlist identifiers \cite{youtube_data_api_docs}.

For comment collection, two resources are especially important. The \texttt{commentThreads} resource supports \texttt{commentThreads.list} and \texttt{commentThreads.insert}. The \texttt{commentThreads.list} method is used to retrieve the comment threads associated with a video, including top-level comments and, in some cases, replies. However, the documentation states that a \texttt{commentThread} does not necessarily contain all replies. For that reason, the \texttt{comments} resource is also relevant. It supports methods such as \texttt{comments.list}, \texttt{comments.insert}, \texttt{comments.update}, \texttt{comments.delete}, and \texttt{comments.setModerationStatus}. In practice, \texttt{comments.list} is required when the system needs to retrieve replies completely rather than relying only on the partial reply information returned within a thread \cite{youtube_data_api_docs}.

The API also includes playlist-related resources that can be useful when data collection begins from a channel instead of a single video. The \texttt{playlists} resource represents playlists and the \texttt{playlistItems} resource represents the items contained within them. In particular, \texttt{playlistItems.list} can be used to enumerate the videos associated with a playlist, including special channel-related playlists such as the uploaded videos list. This can support alternative collection workflows in which videos are first identified from a channel and then analyzed individually \cite{youtube_data_api_docs}.

Although the API provides several write and management methods, this thesis is focused on data retrieval and analysis. Therefore, the implementation centers on read-oriented methods, especially \texttt{search.list}, \texttt{videos.list}, \texttt{channels.list}, \texttt{commentThreads.list}, \texttt{comments.list}, and, when necessary, \texttt{playlistItems.list} \cite{youtube_data_api_docs}.
